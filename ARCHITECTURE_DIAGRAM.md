## Key Patterns & Features

### ğŸ”· **Outbox Pattern (Distributed Transactions)**
- **Problem**: Saving aggregate + publishing events not atomic
- **Solution**: Save aggregate + event to Outbox in single transaction
- **Flow**: 
  1. Entity changes â†’ Event created â†’ Saved to OutBox
  2. Background service polls OutBox every 30s
  3. Publishes events asynchronously with retry logic

### ğŸ”· **Concurrency Control**
- **RowVersion** on Account: `IsRowVersion()` + `IsConcurrencyToken()`
- Prevents lost updates when multiple users modify same account
- Throws `DbUpdateConcurrencyException` â†’ mapped to `ConcurrencyException`

### ğŸ”· **Soft Delete (Logical Delete)**
- **Global Query Filter** auto-excludes deleted records
- `WHERE IsDeleted == false` applied to all queries
- Preserves audit trail

### ğŸ”· **Value Objects**
- `CustomerId`, `AccountId`, `TransactionId`: Strongly-typed IDs
- `AccountNumber`: String wrapped as Value Object
- `Money`: Amount + Currency (Owned Type in EF)

### ğŸ”· **Domain Events**
- Emitted when business logic changes
- Persisted in Outbox table as JSON
- Eventually processed & published to handlers
